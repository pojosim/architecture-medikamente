# Задание 3. Оценка Data Encryption at Rest and In Transit

Вам необходимо разработать стратегию защиты данных в рамках всей системы. При этом важно учесть особенности шифрования данных, которые находятся в разных состояниях — при хранении и во время передачи. 

### Что нужно сделать

1. Определить, какие данные нужно защитить. 
   Необходимо классифицировать информацию по состоянию, чувствительности, профилю риска и другим параметрам. Это поможет выбрать наиболее эффективные методы защиты.
2. Реализовать базовые инструменты защиты. 
   Предложить инструменты защиты, которые нужно добавить в систему, чтобы обеспечить защиту данных.
3. Использовать автоматизированный контроль. 
   Предложить средства для контроля за данными
4. Подготовить реестр программных и аппаратных средств для защиты данных.

В результате подготовьте классификацию данных, обоснование необходимости защиты выделенных классов и инструменты защиты данных.

### Решение 

| Данные                               | В чём необходимость защиты                                | Инструменты защиты                                                                                                    |
|--------------------------------------|-----------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------|
| Медицинские карты, диагнозы, анализы | Врачебная тайна (323-ФЗ), специальные ПД (152-ФЗ)         | AES-256 шифрование при хранении, TLS 1.3 при передаче, доступ на основе ролей RBAC/ABAC, аудит доступа, обезличивание |
| ФИО., адрес, телефон, email          | Персональные данные (152-ФЗ)                              | AES-256 шифрование при хранении, TLS 1.3 при передаче, маскирование, применение DLP-систем                            |
| Платёжные данные                     | Финансовые риски, мошенничество, требования 161-ФЗ        | AES-256 шифрование при хранении, TLS 1.3 при передаче, токенизация                                                    |
| Учётные записи, пароли               | Компрометация ведёт к полной потере контроля над системой | Хеширование (bcrypt), KMS для секретов (Vault), 2FA/MFA                                                               |
| Данные для аналитики (BI/ML)         | Обезличенные данные могут быть реидентифицированы         | Агрегация, динамическое маскирование, контроль доступа к витринам                                                     |
| Данные в резервных копиях            | Утечка резерных копий                                     | AES-256 шифрование резервных копий, контроль доступа к бэкапам                                                        |
| Логи                                 | Содержат метаданные и историю о доступах                  | AES-256 шифрование при хранении, ограниченный доступ, хранение в отдельной системе логирования                        |
